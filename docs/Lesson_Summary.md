# 수리 전문가를 위한 안드로이드 정비 지침서 (Ch.7 요약)

오늘 배운 코드를 기계 정비와 수리의 관점에서 쉽게 정리해 드립니다. 안드로이드 앱도 하나의 정밀한 기계라고 생각하면 이해하기 쉽습니다.

---

### 1. ViewBinding: 대시보드와 엔진 연결하기
*   **코드**: `binding = ActivityTest7Binding.inflate(layoutInflater)`
*   **수리공의 해석**: 기계 외부의 조작 패널(XML 레이아웃)과 내부 제어 장치(Kotlin 코드)를 전선으로 하나하나 연결하는 작업입니다. 예전에는 전선을 일일이 찾았지만(`findViewById`), 이제는 **전용 커넥터(Binding)** 하나만 꽂으면 모든 버튼과 램프를 즉시 제어할 수 있습니다.

### 2. OnClickListener: 스위치 배선 작업
*   **코드**: `binding.btnReserve.setOnClickListener { ... }`
*   **수리공의 해석**: 특정 버튼에 '작동 배선'을 연결하는 것입니다. "이 스위치를 누르면 이 모터가 돌아가게 해라"라는 명령을 내리는 것과 같습니다.

### 3. Date & Time Picker: 타이머 및 스케줄러 설정
*   **코드**: `DatePickerDialog(...)`, `TimePickerDialog(...)`
*   **수리공의 해석**: 기계에 작동 시간을 예약하는 장치를 다는 것입니다. 
    *   **Calendar**: 현재 표준시를 알려주는 기준 시계입니다.
    *   **Picker**: 사용자가 다이얼을 돌려 특정 날짜나 시간을 맞추는 '조절 노브'라고 보시면 됩니다.

### 4. Validation (검사): 가동 전 안전 점검
*   **코드**: `if (name.isBlank()) { ... return@setOnClickListener }`
*   **수리공의 해석**: 기계를 가동하기 전(예약 실행 전)에 연료는 있는지, 안전핀은 뽑혔는지 확인하는 **사전 점검** 단계입니다. 필수 항목이 빠졌다면 사고(에러)를 막기 위해 즉시 가동을 중단(`return`)시킵니다.

### 5. NotifyError: 경고등과 진동 알람
*   **코드**: `Toast.makeText(...)`, `Vibrator.vibrate(...)`
*   **수리공의 해석**: 기계에 문제가 생겼을 때 작업자에게 알리는 방식입니다.
    *   **Toast**: 계기판에 짧게 뜨는 경고 메시지입니다.
    *   **Vibration**: 작업자가 소리를 못 들을 수도 있으니 손에 전달되는 **진동 피드백**을 주는 것입니다. (마치 엔진 과부하 시 핸들이 떨리는 것과 같습니다.)

---

**정비 팁**: 코드가 복잡해 보일 때는 "이 부품이 기계의 어느 부분에서 어떤 신호를 주고받는가?"를 먼저 생각하면 훨씬 명확해집니다. 추가로 궁금한 '부품'이 있다면 언제든 물어보세요!
